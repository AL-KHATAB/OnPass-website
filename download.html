Project distribution

Get the desktop app and extension from the current project repositories.

The active OnPass builds are managed through GitHub. Use the desktop app as the core product and the browser extension as the companion autofill client.

OnPass Desktop

The Windows application that stores the encrypted vault, manages authentication, and serves the localhost API for the extension.

Platform: Windows desktop

Includes: vault management, TOTP, password generation, import/export, biometric flow

Open desktop repo

OnPass Extension

The browser companion for login detection, domain-ranked suggestions, and autofill backed by the local desktop app session.

Platform: Chromium-based browsers

Includes: popup search, token handshake, inline autofill, localhost-only credential retrieval

Open extension repo

Recommended setup flow

1. Get the desktop app

Clone or download the OnPass Desktop project, build it, and run the Windows client first. The desktop app is required before the extension can retrieve credentials.

2. Create or import your vault

Sign in, create your user profile, or import an existing OnPass export. Populate the vault with credentials you want available in the browser.

3. Load the extension

Open the extension repository, load it into your browser, and connect it to the desktop app. The extension uses localhost ports 9876 and 9877 to validate the desktop session.

4. Test autofill

Open a login page, click on a username or password field, and verify that an inline popup appears with ranked credential suggestions.

System requirements

Desktop app

Windows 10 or Windows 11, a working .NET / Visual Studio environment for source builds, and local file access for encrypted vault storage.

Browser extension

A Chromium-based browser with developer mode available so the extension can be loaded locally during development or assessment.

Local API access

The browser and desktop app must run on the same machine so the extension can reach the localhost API exposed by the signed-in desktop session.
