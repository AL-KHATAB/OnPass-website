Privacy assumptions

OnPass is designed around local control of sensitive data.

This page describes the practical privacy assumptions behind the project website, the desktop vault, and the extension's localhost integration model.

Privacy Policy

OnPass is built with a local-first security model. The desktop application stores vault data on the user's device and performs encryption and decryption locally.

The browser extension does not use a remote vault service. Instead, it connects to the signed-in desktop app over localhost and requires a bearer token before credential data is returned.

What OnPass is designed to do

  • Store credential data locally on the machine running the desktop app.
  • Protect saved vault content with encryption before it is written to disk.
  • Expose credentials to the browser extension only through localhost endpoints on the same machine.
  • Keep the browser extension dependent on an active desktop session rather than a remote online account.

What this website does not do

  • It does not provide a cloud dashboard for viewing your vault.
  • It does not submit contact messages to the old third-party Formspree endpoint used by the previous version of this site.
  • It does not host or process your password data.

User responsibility

You remain responsible for safeguarding your master password, exported user files, and any temporary copies created during backup or migration.

Policy changes

If the project architecture changes materially, this page should be updated so the website continues to describe the actual behavior of the desktop app and extension.

Privacy in OnPass comes from local storage, local encryption, and local API boundaries rather than remote trust assumptions.
